#Определение Адресация - информация о том, где находятся обрабатываемые командой данные
Возможно следующее расположение
данных:
1. в самой команде (непосредственная
адресация);
2. в регистрах (регистровая адресация);
3. в памяти (адресация в памяти).
Как правило, в оперативной памяти находиться не больше одного операнда
### Непосредственная адресация
При непосредственной адресации операнд может быть число, адресом, кода ASCII, или иметь символьное обозначения
#### Примеры
```
mov AX, 4c00h
;4c00h 16-ричное число
mov DX, offset mas
;смещение массива mas заноситься в DX
mov CX, !
; Тут в регистр заноситься код ASCII ! знака
```
#ОпределениеПоПониманию offset - это как dword ptr помогает нам получить адресс на этапе компиляции.(==Замечение== с регистрами нельзя такое проворачивать!! ну и впринципе ни с чем нельзя проворачивать)
#### Пример
```
.data ; сегмент данных
 MyVar dw 10
 .code 
 mov ax, offset MyVar
```
### Регистровая адресация
При регистровой адресаци значение из одного регистра переноситься в другой
#### Пример
```
mov AX,BX
;значение BX заноситься в AX
```
==Замечание== 
Регистры разных размеров нельзя заносить!!(ошибка исполнения будет)
### Адресация памяти
Выделяют два способа адресации данных:
#### Прямая
При прямой адресации данных в команде указывается
символическое обозначение ячейки памяти, над
содержимым которой требуется выполнить операцию.
##### Пример
```
mov DL,mem1
; Содержимое байта памяти
с символическим именем mem1
пересылается в DL
```
#### Косвенная Адресация

При косвенной адресации хотя бы одна компонента находится в регистре. При этом имя регистра задаётся в квадратных скобках
##### Пример
```
mov eax,[ebx]
```
