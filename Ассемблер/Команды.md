#командыПересылки Пересылки данных
```
MOV оп1, оп2
переносит содержимое второго операнда (оп2) в первый
операнд (оп1). Содержимое второго операнда не
меняется.
Ограничения, накладываемые командой MOV на
комбинации операндов, следующие:
1. длины операндов должны быть равны;
2. первый операнд не может быть непосредственным;
3. запрещена пересылка память – память;
4. запрещена пересылка в регистры CS и EIP (этим
должны заниматься команды передачи управления);
5. пересылка в сегментные регистры возможна только из
регистров общего назначения или из памяти. 

XCHG оп1, оп2
В соответствии с названием (Xchange) меняет операнды
Пример:
XCHG bl, bh ; верно, но если операнды разной длины то не будет верным
XCHG ax, bl ; неверно.

LEA регистр, адрес
;LEA - это команда позволяющая грузить в регистры адреса на переменные
Пример:
.data
d2 dw 30
.code
mov eax, offset dw; Самое важное это то, что offset грузит эффиктивный адресс а если мы напишем mov eax, offset bx то это будет ошибка
то-есть LEA eax, dw тождевственны с mov eax, offset dw
Примечание: Дальний адресс это сегмент+смещение(дальний адресс не используется в данный момент из-за плоской модели памяти:) )
LDS регистр, дальний_адрес
пересылает смещение из второго операнда в регистр, а адрес сегмента в регистр DS
LES --//---//---// 
LFS --//---//---//
LGS --//---//---// все эти команды просто пересылают в другие регистры ES,FS,GS

```
